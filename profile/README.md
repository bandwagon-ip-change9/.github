## 一、先弄清楚：你的IP是真被封了，还是别的问题？

别急着换IP，先确认一下。很多时候连不上是因为本地网络、端口问题，或者VPS内核崩了——跟IP没关系。

**官方检测方法（最权威）**：

1. 登录搬瓦工后台（KiwiVM Control Panel）
2. 找到你的VPS实例，进入管理页面
3. 访问 `https://kiwivm.64clouds.com/main-exec.php?mode=blacklistcheck`
4. 点击 **Test Main IP**

结果只有两种：

- `IP NOT BLOCKED` → 你的IP没被封，问题出在别处
- `IP BLOCKED` → 确认被封，需要换IP

也可以用第三方工具 [ping.pe](https://ping.pe) 从全球多个节点 ping 一下你的IP，看看国内是否可达。两个工具配合用，准确率更高。

## 二、搬瓦工修改IP的方法有哪些？

### 方法一：免费——切换机房（IP没被封时才管用）

这是最常被忽视的骚操作。搬瓦工每个套餐都支持在多个机房之间免费一键切换，**换机房就等于换IP**。

操作路径：KiwiVM 控制面板 → **Migrate** → 选择目标机房 → 确认

几点要注意：

- **IP没被封才能用**。如果IP已经进了封禁名单，系统会直接提示"Migration backend is currently not available"，这条路堵死了
- 换回原来的机房，IP大概率还是原来那个（除非被别人占用了）
- 切换机房会消耗流量（迁移过程本身会用掉一些）
- 不同套餐支持切换的机房数量不一样，KVM基础套餐可切换约9个机房，CN2 GIA-E套餐多达14个机房

如果你用的是基础KVM套餐，IP还没被封，那这个方法完全免费且实用——相当于白送你9个备用IP。

### 方法三：等待自然解封（免费，但要运气）

如果不急，可以什么都不做，等着瞧。

一般来说，被封的IP经过 **1到3个月**后可能自动解封。原理是GFW那边的封禁名单会定期清理，或者同一个IP段重新被分配后黑名单记录失效。

重装系统或者关机静置，并不会加快解封速度——IP被封是网络层面的事，跟VPS里面跑什么系统无关。

这个方法适合：不急用、能接受长时间等待的用户。
